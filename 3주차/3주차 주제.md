* #### 브루트포스

  1. 개념

     * 가능한 모든 경우를 하나씩 전부 시도해서 정답을 찾는 방식
  2. 필요 이유

     * 암호학에 중요한 판단 기준 하나를 이해 가능

       * 암호가 안전하려면 가능한 키의 수가 충분히 많아야 함
  3. 실제 활용 예시

     1. 카이사르 암호 해독
     2. 비밀번호 추측 공격
     3. PIN 번호 공격
     4. 암호 키 탐색
  4. 이전 개념과의 연결

     * 키의 중요성을 보여주는 공격 예시



* #### 빈도 분석

  1. 개념

     * 암호문에 자주 등장하는 글자를 보고 원래 평문의 글자를 추측하는 공격 방식
  2. 필요 이유

     * 단순 치환 암호가 약한 이유 이해 가능

       * 글자의 통계적 특징이 잔재 -> 빈도 분석에 취약
  3. 실제 활용 예시

     1. 단순 치환 암호 해독
     2. 카이사르 암호 해독 보조
     3. 고전 암호 분석 학습
     4. 언어별 문자 패턴 비교
  4. 이전 개념과의 연결

     * 단순 치환 암호의 한계를 보여줌



* #### 키 공간이 작다는 의미

  1. 개념

     * 공격자가 시도해볼 수 있는 키 후보의 개수가 적음 -> 모든 경우를 직접 확인할 수 있음
     * 키 공간 : 가능한 키의 전체 개수
  2. 필요 이유

     * 암호의 안전성을 판단하는 기준 중 하나가 키 공간의 크기

       * 하지만 '키 공간이 크다 = 무조건 안전하다'는 X
  3. 실제 활용 예시

     1. 카이사르 암호
     2. 숫자 PIN
     3. 비밀번호 길이
  4. 이전 개념과의 연결

     * 키가 중요하다 + 가능한 키가 얼마나 많은지도 중요하다
