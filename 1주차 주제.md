* #### 암호학이 필요한 이유

  * 암호학은 단순히 정보를 숨기는 기술이 아니라, 디지털 사회에서 신뢰를 가능하게 하는 핵심 기술이다. 공개 네트워크에서는 도청, 변조, 위조가 가능하기 때문에, 암호학은 기밀성·무결성·인증·부인방지 같은 보안 기능을 제공한다. 실제로 TLS 1.3은 도청과 변조, 메시지 위조를 막기 위해 설계되었고, NIST는 전자서명이 데이터 무결성과 서명자 인증, 부인방지를 제공한다고 설명한다. 또한 Diffie와 Hellman은 이미 1976년에 키 분배와 인증 문제가 현대 통신 환경의 핵심 장벽이라고 지적했다. 따라서 암호학은 온라인 금융, 전자문서, 소프트웨어 배포, 개인정보 보호 같은 현대 디지털 활동을 가능하게 하는 필수 기반 기술이라고 볼 수 있다.



* #### 기밀성, 무결성, 인증, 부인방지

  * 기밀성 : 허가되지 않은 사람이 정보를 볼 수 없게 하는 것
  * 무결성 : 정보가 허가 없이 바뀌지 않았음을 보장하는 것
  * 인증 : 정보의 출처나 상대방의 신원이 진짜인지 확인하는 것
  * 부인방지 : 어떤 행위를 한 사람이 나중에 그것을 부인하지 못하게 하는 것



* #### 평문, 암호화, 키

  * 평문 : 암호화되기 전, 사람이 그대로 이해할 수 있는 원래 데이터
  * 암호화 : 평문을 암호문으로 변환하는 과정
  * 키 : 암호화/복호화 같은 암호 연산을 제어하는 값
