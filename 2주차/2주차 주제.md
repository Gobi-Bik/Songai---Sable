* #### 카이사르 암호

  * 정의

    * 알파벳 순서를 기준으로 문자를 일정한 수만큼 이동시켜 암호문을 생성
  * 과정

    1. 평문에서 한 글자를 가져온다
    2. 그 글자가 알파벳인지 확인한다
    3. 알파벳이면 정해진 키만큼 이동한다
    4. Z를 넘어가면 다시 A부터 이어서 계산한다
    5. 공백, 숫자, 특수문자는 그대로 두거나 규칙에 따라 처리한다
  * 필요 이유

    * 카이사르 암호는 실제 보안에 사용하기에는 너무 약하지만, 암호학을 처음 배울 때 매우 중요
    * 암호학의 기본 구조를 가장 단순한 형태로 보여주기 때문
  * 실제 활용 예시

    1. 고대 군사 메시지 보호
    2. 암호학 입문 교육
    3. 퍼즐과 게임
    4. ROT13



* #### 단순 치환 암호

  * 정의

    * 평문의 각 문자를 정해진 다른 문자로 바꾸는 고전 암호 방식
  * 과정

    1. 평문에서 한 글자를 읽는다
    2. 그 글자가 치환표에 있는지 확인한다
    3. 치환표에 따라 대응되는 암호문 글자로 바꾼다
    4. 공백, 숫자, 특수문자는 그대로 두거나 정한 규칙에 따라 처리한다
    5. 모든 글자에 대해 반복한다
  * 핵심

    * 각 글자마다 고정된 대응 관계 존재
  * 카이사르 암호와의 비교

|**구분**|**카이사르 암호**|**단순 치환 암호**|
|-|-|-|
|방식|알파벳을 일정 칸 이동|각 알파벳을 임의의 다른 알파벳으로 치환|
|키|이동 칸 수|전체 치환표|
|가능한 키 수|매우 적음|매우 많음|
|약점|브루트포스에 약함|빈도 분석에 약함|
|관계|단순 치환 암호의 특수한 형태|카이사르 암호보다 일반적인 형태|

* //

  * 실제 활용 예시

    1. 고전 암호 체계
    2. 암호학 교육
    3. 퍼즐, 방탈출, 암호 퀴즈
    4. 빈도 분석 학습



* #### 키의 활용 (in 암호화)

  * 키의 정의

    * 평문을 어떤 방식으로 암호문으로 바꿀지 결정하는 핵심 값
  * 핵심 개념

    * 암호화는 알고리즘과 키가 함께 작동하는 과정
    * 같은 평문과 같은 알고리즘을 사용해도 키가 달라지면 암호문이 달라진다
  * 키의 필요 이유

    * 키가 없다면 암호화 규칙이 고정
    * 한 번 규칙이 알려지는 순간 모든 메시지를 쉽게 해독 가능
  * 키의 역할

    * 암호화 결과 결정

      * 같은 평문이라도 키에 따라 다른 암호문이 만들어짐
    * 복호화 제어

      * 올바른 키가 있어야 원래 평문으로 되돌릴 수 있음
    * 보안성 강화

      * 공격자가 알고리즘을 알아도 키를 모르면 해독이 어려워짐
    * 사용자 구분

      * 같은 알고리즘을 쓰더라도 사용자마다 다른 키를 쓸 수 있음
  * 실제 활용 예시

    1. 로그인 비밀번호 자체가 아니라 암호화 키를 보호하는 경우
    2. 파일 암호화
    3. 메시지 암호화
    4. 인증과 무결성 검증



* #### 이전 주차와의 연결

  * 1주차에서는 평문, 암호화, 키, 기밀성 개념을 배웠다
  * 2주차의 카이사르 암호와 단순 치환 암호는 이 중 기밀성을 보여주는 가장 단순한 예시이다
  * 카이사르 암호에서는 키가 이동 칸 수이고, 단순 치환 암호에서는 키가 치환표이다
  * 따라서 2주차는 1주차에서 배운 “키는 암호 연산을 제어하는 값”이라는 개념을 구체적인 고전 암호로 확인하는 과정이다.
